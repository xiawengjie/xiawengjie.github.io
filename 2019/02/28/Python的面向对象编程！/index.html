Python的面向对象编程！

Veröffentlicht am 2019-02-28 Bearbeitet am 2019-03-02 in 编程知识

面向对象：Object Oriented Programming，简称OOP，既面向对象程序设计。

面向对象程序设计和面向过程程序设计不同：面向过程的程序设计，根据业务的逻辑需要从上往下写代码，往往业务逻辑一复杂，代码量上来就难以维护，然而，面向对象编程将对象的数据和函数绑定到一起，进行封装，外部不影响内部，内部不影响外部，这样就能够快速的开发程序，减少了重复的撰写代码。

类和对象

类是一个抽象的名称，但凡具有某种相同或者类似属性和方法的物种集合都可以称作为类

对象，喂喂喂，想哪去了，这不是给你介绍女朋友诶！对象是类的某一个具体的实例，比如，类有猫类，猫类里面有一只叫做阿花的猫，这只阿花的猫就叫做猫类的对象。

魔法方法

在python中，如果是有xxxx()的，那么就有特殊的功能，因此就叫做魔法方法，接下来把常见的魔法方法简单介绍一下。

init方法:

当调用类进行实例化的时候，系统会自动调用init()方法对实例进行属性的操作。

str方法:

在类中定义了str()方法，当使用print()函数打印对象的时候，会将str()方法中返回的字符串打印出来。

del方法:

在Python中，定义一个对象时会自动调用init()方法，同样的，当我们删除一个对象时，系统也会自动调用一个方法del(),会执行del()中的方法。

new方法:

在Python中，当我们定义了一个对象，Python内部会自动调用new方法，利用object中的new方法实例化一个对象出来，并且返回的时候，利用init方法对实例进行属性操作，new方法属于类方法，init方法属于实例方法。

私有属性与私有方法

在Python中没有C++中的public和private这些关键字来区分公有属性和私有属性，它是以属性的命名方式来进行区分的，如果在属性前面加二个下划线“__”，则表明该属性是私有属性，否则为公有属性，方法也是一样，如果在方法前面加了二个下划线的话，表示该方法是私有方法，否则则为公有方法

为什么要设置私有属性，私有方法呢？因为对象里面有些数据需要保护，不能任意的修改，需要将这些数据封装起来，那么就可以定义私有属性，从外部无法直接调用，只能在类的内部多定义一个方法去调用或者修改私有属性，这样就可以达到保护数据的目的。

继承与多态

继承本质是为了减少代码的重复量，当子类从父类继承时，父类的一切特性子类都可以直接调用，私有属性与私有方法除外~

所谓的多态就是指定义时的类型与运行时的类型不一致，在python中崇尚鸭子类型，什么叫做鸭子类型？就是说走路像鸭子，叫起来也像鸭子，那么它就是鸭子。以前我也不太理解这个概念，后来在学python高级教程的时候，我发现好像任何类型都可以通过构造代码变成迭代器和生成器，无所谓它到底是什么类型，是不是迭代器或者生成器都没有关系，只要是可以通过代码做到的迭代器或者生成器的功能，那么那一整块的代码就是迭代器，那么多态也是同理。
